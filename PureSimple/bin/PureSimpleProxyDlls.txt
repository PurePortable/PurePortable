Загрузчики и основная dll
=============================================================================================================================================

Нужный загрузчик из папки loaders поместить в папку программы и туда же поместить одну из следующих основных dll:

PurePort-0.dll - прокси dll без перехвата реестра.
PurePort-1.dll - прокси dll с виртуальным реестром и хранением реестра в текстовом файле.
PurePort-1xp.dll - то же без перехвата функций, отсутствующих в Windows XP.
PurePort-2.dll - прокси dll с использованием подгружаемого куста реестра.
Основную dll переименовать в PurePort.dll или, в соответствии с разрядностью, в PurePort32.dll или PurePort64.dll.

Если основная dll не найдена, никаких сообщений выдано не будет, программа продолжит работу без портабелизации.

Замечания об отдельных загрузчиках:

version.dll - В рамках борьбы с антивирусами исключены отдельные экспортируемые функции.
version+.dll - Более полный экспорт. Для использования переименовать в version.dll.

Основная dll может использоваться без загрузчика если внедряется в импорт утилитами типа setdll. В этом случае имя dll может быть любым допустимым ОС.

=============================================================================================================================================
Dll для использования без загрузчика, внедряемые через замену системной dll в импорте программы
=============================================================================================================================================

Эти dll внедряются правкой импорта программы.

AdvApi32-1.dll - прокси dll с виртуальным реестром и хранением реестра в текстовом файле, для замены advapi32.dll в импорте.
AdvApi32-2.dll - прокси dll с использованием подгружаемого куста реестра, для замены advapi32.dll в импорте.
Нужную dll переименовать в любое имя по схеме 8.3 и заменить AdvApi32.dll в импорте на это имя.
Замена может производится обычным hex-редактром (в кодировке ansi).

ComDlg32-1.dll - прокси dll с виртуальным реестром и хранением реестра в текстовом файле, для замены comdlg32.dll в импорте.
ComDlg32-2.dll - прокси dll с использованием подгружаемого куста реестра, для замены comdlg32.dll в импорте.
Нужную dll переименовать в любое имя по схеме 8.3 и заменить ComDlg32.dll в импорте на это имя.
Замена может производится обычным hex-редактром (в кодировке ansi).

Kernel32-1.dll - прокси dll с виртуальным реестром и хранением реестра в текстовом файле, для замены kernel2.dll в импорте.
Kernel32-2.dll - прокси dll с использованием подгружаемого куста реестра, для замены kernel32.dll в импорте.
Нужную dll переименовать в любое имя по схеме 8.3 и заменить Kernel32.dll в импорте на это имя.
Замена может производится обычным hex-редактром (в кодировке ansi).

Замечание 1: Стандартное имя PurePort.dll тоже подходит для переименования так как укладывается в схему 8.3.
Замечание 2: В целом имя и расширение не имеет значения и не должны превышать 13 символов (8+3+1 вместе с точкой) и заканчиваться нулевым символом (лишние символы надо обнулить).

=============================================================================================================================================
Dll для использования без загрузчика
=============================================================================================================================================

Это часто используемые dll скомпилированные для работы без загрузчика для уменьшения количества файлов в портабелизируемой программе.

Version-0.dll - прокси dll без перехвата реестра.
Version-1.dll - прокси dll с виртуальным реестром и хранением реестра в текстовом файле.
Version-2.dll - прокси dll с использованием подгружаемого куста реестра.
Нужную dll переименовать в version.dll.

WinMM-0.dll - прокси dll без перехвата реестра.
WinMM-1.dll - прокси dll с виртуальным реестром и хранением реестра в текстовом файле.
WinMM-2.dll - прокси dll с использованием подгружаемого куста реестра.
Нужную dll переименовать в winmm.dll.

=============================================================================================================================================
Файл конфигурации
=============================================================================================================================================

Файл конфигурации должен находиться в папке программы.
Имя файла конфигурации должно:
- совпадать с именем основной dll (сначала ищется с расширением .prefs, затем с расширением .ini);
- иметь имя PurePort (аналогично, сначала ищется с расширением .prefs, затем с расширением .ini).
Если не будет обнаружен ни один из этих файлов, будет выдано предупреждение и завершена работа программы.
